# Workflow Analysis for Game Integration

## 单因子工作流 (17 Steps: 0-16)

### Phase 1: 项目初始化 (Step 0) - 一次性
- K线级别选择 (1d/1h/1m etc)
- 资产池筛选规则 (市值Top N, 流动性Top N)
- 数据三段切分 (IS/VAL/OOS)
- 行情覆盖检查

### Phase 2: 因子配置 (Steps 1-3)
- Step 1: 继承项目配置
- Step 2: 定义预测目标 (fwd_period)
- Step 3: 定义退出规则 (hybrid: min_hold → 反转 → max_hold)

### Phase 3: 信号处理流水线 (Steps 4-7)
- Step 4: 构造原始信号 🔀 (用户自然语言描述因子)
- Step 5: Rolling Z-score 标准化
- Step 6: Clip(±3) 去极值
- Step 7: EWMA 平滑

### Phase 4: 仓位与退出 (Steps 8-9)
- Step 8: 仓位映射 (tanh + 波动率缩放)
- Step 9: 退出规则 & 冷却期

### Phase 5: 回测与验证 (Steps 10-11)
- Step 10: IS超参网格搜索 🔀
- Step 11: VAL验证 🔀

### Phase 6: 分析与结论 (Steps 12-16)
- Step 12: 因子画像 (截面分析)
- Step 13: 全市场汇总
- Step 14: 敏感性检验
- Step 15: 分组分析
- Step 16: 最终结论 → 因子档案卡 🔀

## 多因子合成工作流 (12 Steps: M1-M12)
前置: 至少2个因子通过单因子验证

### Phase 1: 加载与去冗余 (M1-M2)
- M1: 加载因子档案卡 & 接口数据
- M2: 因子去冗余 🔀 (相关性矩阵, 阈值0.7)

### Phase 2: 合成配置 (M3-M5)
- M3: 确定合成方式 🔀 (信号层/仓位层)
- M4: 确定合成权重 🔀 (等权/表现加权/滚动动态)
- M5: 合成综合信号

### Phase 3: 仓位与验证 (M6-M7)
- M6: 仓位映射 & 退出规则
- M7: VAL验证 & 调优

### Phase 4: 最终评估 (M8-M12)
- M8: OOS最终评估 (终极考试, 一次性)
- M9: 全市场汇总
- M10: 敏感性检验 (含权重扰动)
- M11: 与单因子基准对比 🔀
- M12: 最终结论 → 组合档案卡 🔀

## 游戏化映射思路

### 研究员任务类型 → 工作流映射
1. **因子挖掘** → 单因子工作流 (Steps 0-16)
2. **策略回测** → 已有因子的参数搜索和验证 (Steps 10-15)
3. **策略优化** → 多因子合成工作流 (Steps M1-M12)

### 用户交互点 (🔀 标记)
单因子: Step 0(项目配置), Step 4(因子定义), Step 10(超参搜索), Step 11(VAL验证), Step 16(结论)
多因子: M2(去冗余), M3(合成方式), M4(权重), M11(对比), M12(结论)

### 关键游戏化元素
- 项目配置 = 公司战略设定 (一次性)
- 因子档案卡 = 研究成果卡片 (收集机制)
- 组合档案卡 = 终极成就
- IS/VAL/OOS = 训练/验证/考试 (三段式紧张感)
- 🔀 交互点 = CEO决策时刻
